# 文字拾い検索 仕様書

## 位置づけ

この文書は、文字拾い検索の仕様を定義します。公開メインツールとして `src/app/(main)` 配下に画面を置き、辞書検索は既存の `SearchManager` / `DICTIONARIES` と `public/dic/*.dic` を利用します。

## 概要

文字拾い検索は、ユーザーが登録した複数の単語から指定文字数ずつ文字を拾い、構成できる辞書語を一覧表示する機能です。基本動作では登録済みの全単語から 1 文字ずつ拾い、その文字集合で作れる単語を検索結果に含めます。文字順を入れ替えて成立する語、つまりアナグラムも検索対象です。

例:

- 登録語が 3 個、文字幅が最小 1 / 最大 1 の場合、検索対象の候補語は 3 文字です。
- 各登録語から 1 文字ずつ使えるなら、候補語の文字順が登録語の順番と異なっていても結果に含めます。
- 文字幅が最小 0 / 最大 2 の場合、各登録語から 0 文字、1 文字、2 文字のいずれかを拾って構成できる語を検索します。

## ルート

| ルート | 役割 |
|---|---|
| `/character-pick-search` | 文字拾い検索の対話画面 |

メイン導線に出すため、`src/lib/json/features.json` の対象です。

## 辞書と対応言語

既存の辞書定義を使います。

| key | 種別 | 用途 |
|---|---|---|
| `buta` | 日本語 | ひらがな中心の日本語辞書 |
| `cefr` | 英語 | CEFR-J の英単語辞書 |

検索対象辞書は画面上で選択できるようにします。日本語は既存辞書検索と同じ正規化方針に寄せ、カタカナ、全角英数字、大文字小文字の差分で不自然な取りこぼしが起きないようにします。英単語は小文字化して判定します。

## 登録語仕様

- ユーザーは単語を複数登録できます。
- 登録できる単語は最大 10 個です。
- 入力値は前後空白を除去して登録します。
- 空文字は登録できません。
- 同じ表示文字列の単語は複数登録できます。重複登録された単語は別々の文字供給元として扱います。
- 登録済み単語は一覧表示します。
- 登録済み単語は削除できます。
- 削除後は検索結果を再計算します。

## 削除 UI

削除操作は、登録済み単語の行を横方向にスワイプすると削除ボタンが現れ、削除ボタンをタップすると単語が削除される UI を基本とします。

実装時の要件:

- モバイルではスワイプ操作を主導線にします。
- PC とキーボード操作では、行のフォーカス時または補助アクションとして削除操作に到達できるようにします。
- 削除ボタンは誤操作を避けるため、スワイプだけで即削除せず、ボタンタップで確定します。
- 10 件以下の短いリストなので、削除後の並び替えや仮想スクロールは不要です。

## 文字の拾い方

文字の拾い方は、各登録語から拾う文字数の幅として指定します。

| 項目 | 初期値 | 説明 |
|---|---:|---|
| 最小 | 1 | 各登録語から拾う最小文字数 |
| 最大 | 1 | 各登録語から拾う最大文字数 |

入力制約:

- 最小、最大はいずれも 0 以上の整数です。
- 0 を指定できます。
- 最小が最大を超える設定は無効です。
- 登録語の長さより大きい文字数は、その登録語では実質的に選べません。最小が登録語の長さを超える場合、その登録語を含む検索条件は該当なしになります。

基本動作:

- 初期状態は最小 1 / 最大 1 です。
- 登録済みのすべての単語から、指定幅の範囲で文字を拾います。
- 初期状態では、登録済みのすべての単語から必ず 1 文字ずつ拾います。
- 最小 0 を指定した場合、その登録語から 1 文字も拾わない構成も許可します。

## 検索判定

検索は「候補語が登録語から拾える文字で構成できるか」を判定します。候補語の文字順は問いません。

候補語が結果に含まれる条件:

1. 候補語が選択中の辞書に存在する。
2. 候補語の正規化後の文字数が、各登録語の文字幅から作れる合計文字数の範囲に入っている。
3. 候補語の各文字を、登録語ごとの文字数制約を満たすように割り当てられる。
4. 各登録語から使う文字数が、指定された最小以上、最大以下である。

この判定により、登録語順に並べた文字列だけでなく、アナグラムとして成立する辞書語も検索結果に含めます。

## 画面要素

| 要素 | 役割 |
|---|---|
| 単語入力欄 | 登録する単語を入力する |
| 単語追加ボタン | 入力中の単語を登録する |
| 登録済み単語一覧 | 登録語を表示し、スワイプ削除操作を提供する |
| 文字の拾い方オプション | 最小 / 最大の文字幅と辞書を選択する |
| 検索結果一覧 | 構成できる辞書語を表示する |
| 結果なしメッセージ | 結果が 0 件の場合に表示する |

検索結果は、登録語の追加、削除、文字幅、辞書の変更に連動して更新します。明示的な検索ボタンは持ちません。

## 状態とメッセージ

| 状態 | 表示 |
|---|---|
| 登録語が 0 件 | 単語を追加する案内を表示し、検索結果は表示しない |
| 空文字を追加しようとした | `単語を入力してください` |
| 登録語が 10 件の状態で追加しようとした | `登録できる単語は10個までです` |
| 最小が最大を超えている | `最小は最大以下にしてください` |
| 検索結果がない | `該当する単語がありません` |
| 辞書読み込み中 | ローディング表示 |
| 辞書読み込み失敗 | 既存辞書検索と同等のエラーメッセージ |

## 表示方針

- 既存の NAZOMATIC のダークグラデーションと `purple-400` アクセントを維持します。
- 文字入力系コントロールはモバイル時 16px 以上にします。
- 画面はモバイルを起点に、入力、登録語、オプション、結果の順で縦に流します。
- 長い説明文は常時表示せず、必要に応じてヘルプ、折りたたみ、ツールチップに逃がします。
- 結果一覧は単語を主表示にし、必要に応じて使用元の内訳を補助表示します。

## 実装方針

文字拾い検索専用の判定ロジックは `src/lib/character-pick-search.ts` に置きます。UI コンポーネントは `src/components/character-pick-search/character-pick-search.tsx` に置き、辞書のロードと正規化は既存の `SearchManager` に寄せます。

候補語を作るために全文字組み合わせを先に列挙すると、登録語が増えたときに組み合わせ数が膨らみます。そのため、辞書語を候補として走査し、各候補語が登録語の文字制約を満たすかを判定する方式を基本にします。

結果数には `src/lib/character-pick-search.ts` の `CHARACTER_PICK_RESULT_MAXCOUNT = 200` を上限として使います。上限に達した場合は、先頭 200 件のみ表示して上限到達の補足を表示します。

## 検証観点

- 空文字を登録できない。
- 10 件を超えて登録できない。
- 登録済み単語を一覧表示できる。
- スワイプ後の削除ボタンから削除できる。
- 初期値の最小 1 / 最大 1 で、全登録語から 1 文字ずつ使う結果だけが出る。
- 最小 0 を指定した場合、特定の登録語から文字を拾わない結果も出る。
- 最大 2 以上を指定した場合、同じ登録語から複数文字使う結果が出る。
- アナグラムが検索結果に含まれる。
- 日本語辞書と英語辞書を切り替えて検索できる。
- 結果がないときに `該当する単語がありません` を表示する。
- モバイルで入力欄が 16px 以上になっている。
